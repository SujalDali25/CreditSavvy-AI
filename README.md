# CreditSavvy AI 💳🤖  
### AI-Powered Credit Card Rewards & Points Optimization System (India)

CreditSavvy AI is a production-style fintech chatbot that helps users **maximize credit card rewards and optimize reward point redemption** using a hybrid system of:

- Deterministic reward computation engine (Python)
- Structured credit card dataset (sample database generated by an LLM)
- LLM-based explanation layer (Google Gemini API)
- FastAPI backend deployed on Render

---

## 🚀 Project Overview

Users can input natural language queries like:

- *“I am booking flights for ₹15,000”*
- *“Which card is best for dining spends?”*
- *“I have 25,000 points, how should I use them?”*

The system responds with:

- Best credit card recommendations
- Reward breakdown and comparisons
- Optimized redemption strategies
- Clear AI-generated explanations

---

## 🧠 Core Idea

The system separates **decision-making** into two layers:

### 1. Deterministic Engine (Core Logic)
- Loads credit card data from dataset
- Computes reward earnings mathematically
- Applies caps, exclusions, and category mapping
- Ranks credit cards objectively

### 2. AI Explanation Layer (Gemini API)
- Converts structured results into human-readable insights
- Explains recommendations clearly
- Handles intent classification and category mapping

---

## 📦 Dataset

The project uses a structured JSON dataset (a **sample database generated by an LLM** to demonstrate functionality).

**Credit Card Database** contains:

- Credit card names
- Reward rates by category (dining, travel, groceries, etc.)
- Caps and exclusions
- Redemption rules (if applicable)

⚠️ This dataset is the **single source of truth** for the engine, but it is a synthetic sample – not real-world financial data.

## 🌐 API Endpoints

### 1. `/chat` (Main Endpoint)  
Handles natural language queries.

**Request:**  
json

  "message": "I am booking flights for 15000"
}

Response:

Top 3 credit cards

Reward breakdown

Recommendation

Explanation

2. /recommend
Returns best credit cards for a spending category.

3. /redeem
Optimizes reward point usage strategy.

4. /compare
Compares two credit cards based on reward performance.
